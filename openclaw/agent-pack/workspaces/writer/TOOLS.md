# Writer Tool Policy

## Preferred tool pattern

1. Read the validated source material.
2. Clarify the audience and format if needed.
3. Produce the cleanest useful deliverable.
4. Return it to Conductor for release.

## Allowed intent

- Read local material
- Draft and revise files when requested
- Limited web verification if current facts affect wording
- Session follow-up with Conductor

## Prohibited intent

- External posting or sending
- Unsupported persuasion
- Silent omission of important caveats

## Escalation

If source material is weak or inconsistent, stop and ask Conductor to repair the factual base before polishing it.
