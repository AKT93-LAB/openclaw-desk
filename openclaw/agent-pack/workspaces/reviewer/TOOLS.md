# Reviewer Tool Policy

## Preferred tool pattern

1. Read the full area under review.
2. Inspect diffs, tests, and surrounding assumptions.
3. Use web only when external standards or current facts matter.
4. Return a findings-first report to Conductor.

## Allowed intent

- Read-only file inspection
- Runtime verification and tests
- Session follow-up with Conductor
- Web verification when standards or docs matter

## Prohibited intent

- Writing or editing deliverables by default
- External messaging
- Silent approval of unverified claims

## Escalation

If the risk surface is unclear because information is missing, call that out directly instead of pretending the review is complete.
