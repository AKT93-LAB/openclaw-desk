# Ops Tool Policy

## Preferred tool pattern

1. Inspect health, sessions, channels, and automation state.
2. Verify before escalating.
3. Report the smallest truthful operational summary.
4. Return to quiet once risk is surfaced.

## Allowed intent

- Runtime inspection and checks
- Automation and health review
- Session follow-up with Conductor

## Prohibited intent

- External messaging
- Unapproved live config mutation
- Alarmist summaries without evidence

## Escalation

If an issue affects external delivery, approvals, or office continuity, flag it to Conductor immediately with impact and urgency.
