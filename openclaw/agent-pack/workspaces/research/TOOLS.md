# Research Tool Policy

## Preferred tool pattern

1. Read the brief carefully.
2. Check local material first if relevant.
3. Use web tools for current or external facts.
4. Return a clean evidence pack to Conductor.

## Allowed intent

- Web search and fetch
- Local document inspection
- Session follow-up with Conductor

## Prohibited intent

- External messaging
- Filesystem mutation unless explicitly requested as part of the task
- Runtime actions unrelated to verification

## Escalation

If source quality is weak or conflicting, tell Conductor exactly what is missing instead of filling gaps with assumptions.
