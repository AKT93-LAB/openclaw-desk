ClawDesk Presentation

ClawDesk for OpenClaw

ClawDesk is a chat-first mission control layer for OpenClaw. It gives one operator a human control room for Nova, live office activity, approvals, automations, and agent editing without requiring direct config-file work for day-to-day administration.

Zero mock policy. Same-host deployment. Review-first writeback.

Product shape

What ClawDesk is

User experience

  • Nova is the default entry point.
  • The operator sees a clean office-style dashboard.
  • Internal agent activity is summarized for humans, with advanced details available.

Technical shape

  • Browser UI in front.
  • Local ClawDesk service on the same host as OpenClaw.
  • OpenClaw gateway bridge plus local workspace file access behind the UI.

Architecture

How ClawDesk works

Browser

Shows the shell, chat, live work, agent editor, automation summary, approvals, and settings.

ClawDesk service

Holds gateway credentials server-side, builds dashboard snapshots, proxies events, and performs writeback.

OpenClaw host

Supplies real runtime data and real local workspace files. If this side is missing, ClawDesk stays empty and honest.

Install path

Recommended deployment flow

Host setup

  • clone repo to /opt/clawdesk
  • set env in /etc/clawdesk/clawdesk.env
  • build with npm run build
  • run behind systemd on 127.0.0.1:3010

Live enablement

  • verify gateway connection
  • verify /api/dashboard returns mode: "live"
  • review and merge the office pack safely
  • validate Nova, approvals, events, agent editing, and file editing

Current proof

What has been verified already

  • typecheck and production build pass
  • the production app serves on 127.0.0.1:3010
  • /api/dashboard returns the strict offline snapshot when no gateway is configured
  • desktop shell and mobile drawers were verified with Playwright

Live OpenClaw behavior still requires VPS-side validation because ClawDesk will not fake that data.
